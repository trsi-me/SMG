# دليل إعداد الهاردوير - للمبتدئين
# Hardware Setup Guide - For Beginners

## 📦 ما تحتاجه (المكونات)

### المكونات الأساسية:

1. **Arduino Uno** أو **ESP32**
   - **ما هو؟**: لوحة إلكترونية صغيرة (مثل دماغ المشروع)
   - **مثله مثل**: معالج الكمبيوتر لكن أبسط
   - **وظيفته**: يقرأ البيانات من الحساسات ويرسلها للكمبيوتر

2. **شاشة LCD 16x2 مع I2C**
   - **ما هو؟**: شاشة صغيرة تعرض النصوص (16 حرف × سطرين)
   - **مثله مثل**: شاشة صغيرة مثل الموجودة في الآلات الحاسبة
   - **وظيفته**: يعرض نسبة الرطوبة والحرارة على الشاشة
   - **ملاحظة**: تأكد أن الشاشة تحتوي على "I2C" (لوحة صغيرة ملحقة)

3. **حساس رطوبة التربة** (Soil Moisture Sensor)
   - **ما هو؟**: عصا صغيرة تدخلها في التربة
   - **وظيفته**: يقيس كم الماء في التربة (جاف أم رطب)

4. **حساس DHT22**
   - **ما هو؟**: حساس صغير يقيس الحرارة والرطوبة
   - **وظيفته**: يخبرك بدرجة الحرارة ورطوبة الجو

5. **أسلاك توصيل** (Jumper Wires)
   - **ما هي؟**: أسلاك ملونة لربط المكونات
   - **مثلها مثل**: أسلاك الكهرباء لكن أرفع

6. **مقاوم 10K ohm** (اختياري)
   - **ما هو؟**: قطعة صغيرة تساعد الحساس على العمل بشكل صحيح
   - **ملاحظة**: بعض الحساسات تأتي معه مدمج

## 🔌 كيف تربط المكونات (التوصيلات)

### الخطوة 1: ربط شاشة LCD

**شرح المصطلحات:**
- **VCC** = الطاقة الموجبة (مثل +)
- **GND** = الأرضي (مثل -)
- **SDA** = سلك البيانات
- **SCL** = سلك الساعة (للتواصل)

**التوصيلات:**
```
شاشة LCD          →    Arduino
───────────────────────────────
VCC (أحمر)        →    5V
GND (أسود)        →    GND
SDA (أبيض/أخضر)   →    A4 (Arduino Uno) أو GPIO21 (ESP32)
SCL (أصفر)        →    A5 (Arduino Uno) أو GPIO22 (ESP32)
```

**💡 نصيحة**: الألوان قد تختلف حسب الشركة، المهم هو الأسماء (VCC, GND, SDA, SCL)

---

### الخطوة 2: ربط حساس رطوبة التربة

**التوصيلات:**
```
حساس التربة       →    Arduino
───────────────────────────────
VCC (أحمر)        →    5V
GND (أسود)        →    GND
A0 (أبيض/أصفر)    →    A0
```

**💡 نصيحة**: حساس التربة له 3 أسلاك فقط، ربطها بسيط

---

### الخطوة 3: ربط حساس DHT22

**التوصيلات:**
```
حساس DHT22        →    Arduino
───────────────────────────────
VCC (أحمر)        →    5V
GND (أسود)        →    GND
DATA (أبيض)       →    Pin 2
```

**💡 نصيحة**: إذا كان الحساس لا يعمل، أضف مقاوم 10K بين DATA و VCC

## 💻 تثبيت البرنامج (Arduino IDE)

### الخطوة 1: تحميل Arduino IDE
1. اذهب إلى: https://www.arduino.cc/en/software
2. حمّل النسخة المناسبة لنظامك (Windows/Mac/Linux)
3. ثبت البرنامج

### الخطوة 2: تثبيت المكتبات المطلوبة

**ما هي المكتبات؟**
- المكتبات = برامج جاهزة تساعدك في استخدام المكونات
- مثل تطبيقات الهاتف، لكن للـ Arduino

**كيف تثبتها:**

1. افتح Arduino IDE
2. اذهب إلى: **Sketch** → **Include Library** → **Manage Libraries**
   - أو اضغط: `Ctrl+Shift+I` (Windows) أو `Cmd+Shift+I` (Mac)
3. ابحث عن كل مكتبة وثبتها:

   **أ) LiquidCrystal_I2C**
   - ابحث عن: `LiquidCrystal_I2C`
   - اختر: **by Frank de Brabander**
   - اضغط: **Install**

   **ب) DHT sensor library**
   - ابحث عن: `DHT sensor library`
   - اختر: **by Adafruit**
   - اضغط: **Install**
   - **مهم**: عند التثبيت، سيطلب تثبيت مكتبة أخرى (Adafruit Unified Sensor)، اضغط **Install All**

   **ج) ArduinoJson**
   - ابحث عن: `ArduinoJson`
   - اختر: **by Benoit Blanchon**
   - اضغط: **Install**

## 📤 رفع الكود على Arduino

### الخطوة 1: فتح الكود
1. افتح ملف: `hardware/arduino_esp32_code.ino` في Arduino IDE
   - **كيف؟**: File → Open → اختر الملف

### الخطوة 2: اختيار نوع اللوحة

**إذا كان لديك Arduino Uno:**
- اذهب إلى: **Tools** → **Board** → **Arduino AVR Boards** → **Arduino Uno**

**إذا كان لديك ESP32:**
- اذهب إلى: **Tools** → **Board** → **ESP32 Arduino** → **ESP32 Dev Module**
- **ملاحظة**: قد تحتاج تثبيت دعم ESP32 أولاً (Tools → Board → Boards Manager → ابحث عن ESP32)

### الخطوة 3: اختيار المنفذ (Port)

**ما هو المنفذ؟**
- المنفذ = المكان الذي يتصل فيه Arduino بجهاز الكمبيوتر
- مثل منفذ USB لكن له اسم خاص

**كيف تعرفه:**

**في Windows:**
1. اربط Arduino بالكمبيوتر
2. اذهب إلى: **Tools** → **Port**
3. اختر المنفذ الذي يظهر (مثل COM3, COM4)
   - **💡 نصيحة**: إذا لم ترَ أي منفذ، جرب فصل وربط Arduino مرة أخرى

**في Mac/Linux:**
- عادة يكون: `/dev/ttyUSB0` أو `/dev/ttyACM0`

### الخطوة 4: رفع الكود
1. اضغط زر **Upload** (السهم الأيمن) في الأعلى
2. انتظر حتى ترى "Done uploading"
3. **مهم**: لا تفصل Arduino أثناء الرفع!

## ✅ اختبار أن كل شيء يعمل

### الخطوة 1: فتح Serial Monitor

**ما هو Serial Monitor؟**
- نافذة تعرض البيانات التي يرسلها Arduino
- مثل شاشة صغيرة ترى فيها ما يحدث

**كيف تفتحه:**
- اضغط: `Ctrl+Shift+M` (Windows) أو `Cmd+Shift+M` (Mac)
- أو: **Tools** → **Serial Monitor**

### الخطوة 2: التحقق من الإعدادات

في Serial Monitor، تأكد من:
- **Baudrate** = 9600 (في الزاوية السفلية اليمنى)
- إذا لم يكن 9600، غيّره من القائمة المنسدلة

### الخطوة 3: ما يجب أن تراه

**في Serial Monitor:**
- كل 30 ثانية، يجب أن ترى بيانات مثل:
```json
{"device_id":"ESP32-001","soil_raw":450,"temperature_c":25.5,"humidity_percent":60.0}
```

**على شاشة LCD:**
- يجب أن ترى:
```
S:55% T:25.5C
H:60% B:100%
```
- **S** = رطوبة التربة
- **T** = درجة الحرارة
- **H** = رطوبة الجو
- **B** = البطارية

**إذا رأيت هذا، مبروك! 🎉 كل شيء يعمل**

## 🔗 ربط Arduino مع التطبيق (Backend)

### الخطوة 1: تشغيل Backend

1. افتح Terminal/Command Prompt
2. اذهب لمجلد المشروع:
   ```bash
   cd backend
   ```
3. شغّل الخادم:
   ```bash
   python main.py
   ```

### الخطوة 2: معرفة منفذ Arduino

**في Windows:**
- افتح Device Manager
- ابحث عن "Ports (COM & LPT)"
- اكتب رقم المنفذ (مثل COM3)

**أو استخدم الكود:**
- افتح المتصفح واذهب إلى: `http://localhost:8000/api/v1/hardware/ports`
- سترى قائمة بالمنافذ المتاحة

### الخطوة 3: الاتصال بالجهاز

**الطريقة البسيطة (من المتصفح):**
- لا تحتاج هذا الآن، التطبيق سيتصل تلقائياً

**الطريقة المتقدمة (للاختبار):**
- استخدم Postman أو أي أداة مشابهة
- أو استخدم curl (للمستخدمين المتقدمين فقط)

## ❌ حل المشاكل الشائعة

### المشكلة 1: لا تظهر بيانات في Serial Monitor

**الأسباب المحتملة:**
- ❌ Baudrate خاطئ (يجب أن يكون 9600)
- ❌ Arduino غير متصل
- ❌ الكود لم يُرفع بشكل صحيح

**الحل:**
1. تأكد من Baudrate = 9600
2. أعد رفع الكود
3. جرب فصل وربط Arduino

---

### المشكلة 2: شاشة LCD لا تعمل (سوداء أو فارغة)

**الأسباب المحتملة:**
- ❌ التوصيلات خاطئة
- ❌ عنوان I2C خاطئ

**الحل:**
1. تحقق من التوصيلات (خاصة SDA و SCL)
2. جرب تغيير العنوان في الكود:
   - ابحث عن: `LiquidCrystal_I2C lcd(0x27, 16, 2);`
   - غيّر `0x27` إلى `0x3F` (أو العكس)
3. تأكد أن شاشة LCD مضاءة (يوجد زر صغير للسطوع)

---

### المشكلة 3: حساس DHT22 يعطي أرقام غريبة (NaN)

**الأسباب المحتملة:**
- ❌ التوصيلات خاطئة
- ❌ يحتاج مقاوم 10K

**الحل:**
1. تحقق من التوصيلات
2. أضف مقاوم 10K ohm بين DATA و VCC
3. جرب تغيير Pin (من Pin 2 إلى Pin 4 مثلاً)

---

### المشكلة 4: Backend لا يتصل بالجهاز

**الأسباب المحتملة:**
- ❌ Arduino IDE Serial Monitor مفتوح (يجب إغلاقه)
- ❌ المنفذ غير صحيح
- ❌ Backend غير مشغّل

**الحل:**
1. **أغلق Arduino IDE Serial Monitor** (مهم جداً!)
2. تأكد من تشغيل Backend
3. تحقق من رقم المنفذ

## 📚 مصطلحات مهمة (قاموس سريع)

- **Arduino**: لوحة إلكترونية صغيرة قابلة للبرمجة
- **ESP32**: نوع من Arduino لكن أقوى ويدعم WiFi
- **LCD**: شاشة صغيرة لعرض النصوص
- **I2C**: طريقة للتواصل بين المكونات (يحتاج سلكين فقط)
- **Sensor (حساس)**: جهاز صغير يقيس شيئاً (مثل الحرارة)
- **Serial**: طريقة لإرسال البيانات من Arduino للكمبيوتر
- **Baudrate**: سرعة إرسال البيانات (9600 = سرعة قياسية)
- **Port (منفذ)**: المكان الذي يتصل فيه Arduino بالكمبيوتر
- **Upload**: رفع الكود على Arduino
- **VCC**: الطاقة الموجبة (+)
- **GND**: الأرضي (-)
- **Pin**: دبوس في Arduino لربط المكونات
